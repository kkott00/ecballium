Getting started

Let's pretend simple situation:
You have a button on page. If you press this button you get some caption with specified text. Hence you need to test if caption is appeared on button pressing, test their position and test text in this caption.

This is the way we like to resolve such situations:
First of all we create test description using simplified Gherkin language. 

#demo.feature
Feature: Demo

Scenario: Press button
  Find button with text "Big Button"
  Click found item
  Check if there is caption "Big Button pressed"

Next make code description of steps which is not described in framework library. You can use CoffeeScript or Javascript for this. 

#demo.coffee
window.ecb.handlers=window.ecb.handlers.concat [
 [/^Check if there is caption "([^"]+)"/,
  (out)->
   f=$("p.caption:contains(#{out})")
   @assert(f.length==1,'No such caption')
 ]
]

window.ecb.aliases.extend 
  caption: 'p.caption'

Finally we just copy files demo.feature and demo.coffee as static files to folder static/test and open page in browser you like to test. You got test results as series or POST requests to URL test. The only server-side code you need is handler for URL test which will collect test results. But if you are really serious you can see POST request in browser debug console.
