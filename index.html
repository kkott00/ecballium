Getting started

Let's pretend simple situation:
You have a button on page. If you press this button you get some caption with specified text. Hence you need to test if caption is appeared on button pressing, test their position and test text in this caption.

This is the way we like to resolve such situations:
First step Create test description using simplified Gherkin language. 

#demo.feature
Feature: Demo

Scenario: Press button
  Find button with text "Big Button"
  Click found item
  Check if there is caption "Big Button pressed"

Second step We just deploy files demo.feature, ecballium.js and config.js as static files (for example in folder test). Also we need add script ecballium.js to head of pages for testing (for example to base template). 

 <!-- Jinja example-->
 {% if config.TEST %}
  <script src="test/ecballium.js"></script>
 {% endif %}

Finally Open page in browser you like to test.
Here you can see some extended demo how it works. You get test results as series or POST requests to URL test. The only server-side code you need is handler for URL test which will collect test results. Also you can see test result in debug console.
